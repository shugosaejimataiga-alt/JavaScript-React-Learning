🟦 Day10：API / JSON の役割を理解（概念だけ）

今日は まだコードを書かない日 です。
フロントエンドとバックエンドが
どうやって安全に会話しているのか を整理します。



■ まず超シンプルに説明すると…

API ＝ フロントとバックエンドの会話用の窓口

JSON ＝ その会話で使うデータの書き方（形式）

フロントとバックエンドは
直接つながってはいない。



■ 本質的に大切な考え方

✔ 1) API は「境界線」である

フロントとバックエンドの間には
明確な境界 がある。

フロントは中身を知らない

バックエンドは画面を知らない

👉 その境界が API



✔ 2) JSON は「外との会話専用の言語」

JSON が使われるのは、

ブラウザ（React） ⇄ Controller（Laravel）

この 境界部分だけ。

フロント ⇄ バックエンド：JSON

Controller 以降（Service / Model）：PHPの配列・オブジェクト

👉 内部処理では JSON は使わない。



✔ 3) フロントは「お願い」、バックエンドは「判断」

フロントは
「この条件でデータをください」と お願い するだけ。

DBを直接操作しない
保存方法を知らない

👉 判断・保存・安全性は
すべてバックエンドの責務。



✔ 4) 責務分離が壊れると、全体が壊れる

もしフロントが

DB構造を前提にしたり
内部処理を知っていたら

👉 バックエンド変更＝フロント崩壊

だから APIで切り分ける 必要がある。



■ 今日いちばん重要な一文

APIはフロントとバックエンドの責務を守る境界線で、
JSONはその境界を越えるための共通言語。

ここまでが Day10の概念 です。
