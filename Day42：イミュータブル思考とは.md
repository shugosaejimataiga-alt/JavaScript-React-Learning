
## 1.✅ Day42｜今回の学習の説明（概念）


**イミュータブル（immutable）**とは、
👉 **「元のデータを直接書き換えない考え方」**です。


JavaScriptでは、

配列
オブジェクト

は 書き換え可能（ミュータブル） ですが、
Reactでは あえて書き換えない のが重要です。



なぜ？

直接書き換えると
👉 Reactが「変わった」と気づけない

結果として
👉 画面が更新されない / バグる

つまり：

「新しいデータを作って置き換える」
これがイミュータブル思考です。

## 2.基本の書き方（超重要）


❌ ダメな例（ミュータブル）

const numbers = [1, 2, 3];
numbers.push(4); // 元の配列を直接変更


✅ 良い例（イミュータブル）

const numbers = [1, 2, 3];
const newNumbers = [...numbers, 4];


ポイント：

numbers は 触らない

newNumbers という 新しい配列を作る




オブジェクトの場合

❌ ダメ：

const user = { name: "Taro", age: 20 };
user.age = 21;


✅ 良い：

const user = { name: "Taro", age: 20 };
const newUser = { ...user, age: 21 };

## 3.練習問題1

次のうち、イミュータブルなのはどれでしょう？

A:
todos.push("勉強");

B:
const newTodos = [...todos, "勉強"];

👉 A / B のどちらかを答えてください。


解答：B


正解です。B ✅


理由も端的に言うとこうです。

A：元の todos を直接変更している（ミュータブル）

B：元の todos は触らず、新しい配列を作っている（イミュータブル）

ここまで完璧です

## 4.練習問題2

次のコードの 空欄 を埋めてください。
※ 元の配列 actions は 絶対に変更しない こと。

const actions = ["run", "study"];

const newActions = _____________;


やりたいこと：

"sleep" を追加した 新しい配列 を作る

👉 コードを書いてください（1行でOK）



解答：

const actions = ["run", "study"];
const newActions =[...actions , "sleep"];



✅ 正解です。完璧。

const actions = ["run", "study"];
const newActions = [...actions, "sleep"];


元の actions はそのまま

新しい配列 newActions を作って追加
👉 イミュータブル思考を正しく使えています

## 5.練習問題3

次は オブジェクト版 です。
やることは同じで、元データは変更しません。

const dream = {
  title: "エンジニアになる",
  progress: 30
};

// progress を 40 にした newDream を作りたい
const newDream = ______________________;


👉 1行で書いてください。



解答：

const newDream = {...dream , progress: 40};


✅ 正解です。これも完璧。

const newDream = { ...dream, progress: 40 };

dream は一切変更していない

新しいオブジェクトを作って progress だけ更新
👉 イミュータブル思考、完全に理解できています。

✅ Day42｜まとめ（今回の学習）

イミュータブル思考とは
👉「元データを直接書き換えず、新しいデータを作る」考え方。



なぜ重要か

React は
👉「参照が変わったか」で再描画を判断する

直接変更すると
👉 変化に気づかず、画面が更新されない



基本パターン

配列：[...array, newItem]

オブジェクト：{ ...obj, key: newValue }
