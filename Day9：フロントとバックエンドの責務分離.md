🟦 Day9：フロントとバックエンドの責務分離

今日は、アプリ開発の中で最も重要な考え方のひとつである
「フロントとバックエンドの役割の違い」 を整理します。

■ 超シンプルに説明すると…

フロント＝見せる仕事
バックエンド＝データを守り、処理する仕事

どちらもアプリには必要ですが、得意なこと・やるべきことがまったく違います。



■ 本質的に大切な考え方

✔ 1) フロントの責務

フロントの役割は “表示” と “操作” です。

ボタン、入力フォーム、一覧画面
状態を管理し、画面に反映する

ユーザーが見える部分すべて

「どう見えるか」「どう動くか」を決める仕事

データそのものを決めるのではなく、
データをどう見せるか がフロントの責務です。



✔ 2) バックエンドの責務

バックエンドの役割は “本物のデータを扱う部分”。

ユーザーを区別する
データを保存する（DB）
ロジックを処理する
セキュリティを保つ
外部に公開してはいけない情報を管理する

つまり、
データの本当の世界を守り、処理するのがバックエンド。



✔ 3) なぜ責務分離が大事なのか

もしフロントが全部やろうとすると…

セキュリティ的に危険
コードが肥大化して壊れやすい
画面変更があるたびに全部作り直し
チーム開発が成り立たない

だから Web アプリは必ずこう分かれます：

フロント → 表示と操作
バックエンド → データ処理と保存

お互いが API という橋を渡して通信する仕組みになります。



■ 今日の理解：あなたの言葉で言うと…

フロントは
「どう見せるか・どう動かすか」 を作る場所。

バックエンドは
「本物のデータを扱い、ユーザーを区別して守る場所」。
