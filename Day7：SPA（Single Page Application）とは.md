Day7：SPA（Single Page Application）とは

今日は、Webアプリが“どう画面を切り替えるか”という仕組みを整理する日です。
ここでもまだコードは書きません。概念だけ理解します。

■ まず超シンプルに説明すると…

SPA＝ページを切り替えずに、中身だけ変えて動くアプリの仕組み。

普通のWebサイト（MPA）は、リンクを押すたびに
「真っ白 → 読み込み → 新しいページ表示」となるのに対し、

SPA は
ページをリロードせずに画面だけ入れ替わる。

だから速くて、アプリのように滑らかに動きます。



■ 本質的に大切な考え方

✔ 1）SPAでは“ページは1枚のまま”

従来のWebサイトはページごとにHTMLを読み直すが、
SPA は 最初の1回だけHTMLを読み込むスタイル。

その後は React（JS）が

“必要な部分だけ差し替える”

ことで画面遷移しているように見せています。



✔ 2）画面切り替えの主役はブラウザではなく JavaScript

従来のページ遷移：
→ ブラウザが新しいページを取得して表示

SPA の遷移：
→ JavaScript（React）がUIを書き換える

これにより再読み込みがなく、高速で UI も壊れにくい。



✔ 3）SPA を理解すると React の役割がわかる

現代のアプリは

一覧を更新
詳細を開く
フォーム編集
戻る
進捗変化

画面が頻繁に変わるため、
毎回ページを読み直す方式（MPA）では限界がある。

→ だからこそ、
“中身だけを賢く差し替える”仕組み（SPA）が必要
→ その仕組みを実現するのが React という流れにつながる。



今日の概念まとめ

SPAとは、
ページを再読み込みせず、React が中身だけを書き換えて動かすアプリ構造。
速くて滑らかで、複雑な画面にも耐えられる現代的なスタイル。
