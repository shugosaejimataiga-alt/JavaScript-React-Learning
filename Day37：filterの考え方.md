
## 1.filter の考え方（役割理解）

① 何を学ぶ日か

今日は 「配列から条件に合うものだけを取り出す考え方」 を学びます。
それを担当するのが filter です。



② filter は何をするもの？

filter = ふるいにかける です。

配列を 1つずつ確認する
条件に 合うものだけ残す

新しい配列 を作る
（元の配列は変わらない）

👉 減らすための道具 です。



③ for文との役割の違い（重要）

for
→ 何でもできるが、毎回「箱・if・push」を書く必要がある

filter
→ 「残す条件」だけを書く

つまり filter は
「配列から選別する」という目的に特化した道具 です。



④ 頭の中のイメージ

「この配列の中から
条件を満たすものだけ欲しい」

このときに filter を使います。

## 2.filter の基本の書き方

① 最小の形↓


const newArray = array.filter(item => 条件);

これだけです。



② 何が起きているか（超重要）

filter の中では、次のことが起きています。

配列の中身を 1つずつ取り出す
↓
item に入れる
↓
条件をチェックする
↓
true → 残す
↓
false → 捨てる
↓
残ったものだけで 新しい配列 を作る



③ 超シンプル例

const numbers = [1, 2, 3, 4];

const result = numbers.filter(n => n > 2);


このとき：

1 → false（捨てる）
2 → false（捨てる）
3 → true（残す）
4 → true（残す）

結果：

[3, 4]



④ 絶対に覚える1点

👉 filter は true / false で残すか決める

## 3.練習問題（手を動かす思考）

次の配列があります。

const ages = [12, 18, 25, 9, 30];

問題

「18歳以上の人だけを残す配列」を作りたいとき、
filter を使ってコードを書いてください。


const agesFilter = ages.filter(age => age >= 18);

Day37｜filter で学んだこと【完全版】


① filter とは何か

filter は配列から「条件に合うものだけ」を取り出すための仕組み。
ふるいにかけて、残すか捨てるかを決める。



② なぜ filter を使うのか

for文でも同じことはできるが、

箱を作る
ifを書く
pushする

といった 余計な手順 が必要になる。

filter は
「選別する」という目的だけに特化しているため、
考えることが少なく、読みやすい。



③ 基本構造（最重要）


配列.filter(要素 => 条件)


配列：必ず 配列の名前
要素：配列から1つずつ取り出した値
条件：true / false を返す式

👉 filter は 配列が持っている機能。



④ filter の中で起きていること

配列の中身を1つ取り出す
↓
条件をチェックする
↓
true → 残す
↓
false → 捨てる
↓
残ったものだけで 新しい配列を作る



⑤ filter の超重要ルール

条件は true / false

元の配列は変わらない

結果は 必ず新しい配列



⑥ よくある間違いと修正

間違い①

array.filter(...)


→ array という配列が存在しないと動かない。


正解

ages.filter(...)



⑦ 条件式の意味理解（超重要）

> ：含まない

>=：含む

「18歳以上」＝ >= 18



⑧ 今回の練習で理解したこと

const agesFilter = ages.filter(age => age >= 18);


これは
「ages 配列の中から、18歳以上のものだけを残す」
という意味を そのままコードにした形。



最終まとめ（核心）

filter は「配列を条件でふるいにかけ、元を壊さずに新しい配列を作るための道具」。
