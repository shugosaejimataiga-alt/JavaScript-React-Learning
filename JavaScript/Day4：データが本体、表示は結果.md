🟦 Day4：データが本体、表示は結果（概念だけ）

今日は フロントエンドの本質に直結する考え方 を整理しました。
まだコードは書きません。

■ まず超シンプルに説明すると…

アプリの本体は「画面」ではなく “データ” です。

画面はただの“結果”であり、
内部にある事実情報（title / date / 重要度 / 完了状態など）が
どう変化したかを映し出しているだけ。



■ 本質的に大切な考え方

✔ 1) UI はデータの「影」

人はつい画面そのものを本体だと思いがちですが、
本体は バックエンドやフロント内部にあるデータ。

夢のタイトル
日付
重要度
完了したかどうか

これらが 「アプリが覚えておくべき事実」＝データ。

UI はその結果にすぎません。



✔ 2) データを直接触らず、画面だけ変えると壊れる

画面だけを書き換えると、

データとUIがズレる
修正が難しくなる
規模が大きくなるほど破綻する

という問題が必ず起こります。

だから 本体（データ）を変える → UIが勝手に変わる
という正しい流れが必要。

Reactが重要になる理由もここ。



✔ 3) バックエンドとフロントのデータは“必要な時だけつながる”

今日の理解で特に重要な部分。

フロントは必要な時にバックエンドからデータを取りに行く

受け取ったら 一時的に自分の状態（State）として保持する

更新するときだけ、またバックエンドへ送る

ずっとリアルタイムに繋がっているわけではなく、
「必要なときだけ通信する」関係。



■ 今日、あなたが理解したこと（まとめ）

データとは「アプリが内部で覚えているべき事実」

title や date、重要度はすべて“本体”を構成するデータ

UIはそのデータを映している“結果”でしかない

フロントはバックエンドから必要な時だけデータを取得し、
自分のStateとして扱う

バックエンドとフロントは“ずっとつながる”のではなく、
必要な瞬間だけ橋が架かる



■ 一言でまとめると…

データが本体。
画面はその結果。
フロントはデータを正しく扱う仕事。




titleやdateなどのデータが本体で、UIはそれをどう見せるかの飾りである
