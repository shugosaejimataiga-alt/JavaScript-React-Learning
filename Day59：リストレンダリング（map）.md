

📘 Day59：リストレンダリング（map）


① Reactのリストレンダリングとは？

データの配列 → JSXの配列 に変換すること

{array.map(item => (
  <p>{item}</p>
))}

mapは
👉 新しい配列を返す

Reactでは
👉 JSXの配列を返している



② 配列の長さ ＝ 表示数

3要素 → 3個表示

10要素 → 10個表示

0要素 → 何も表示されない

Reactは配列の数だけUIを作る。



③ keyが必須

{users.map(user => (
  <p key={user.id}>{user.name}</p>
))}

なぜ必要？

Reactは
どの要素がどれか
を識別する必要がある。



④ indexが危険な理由

indexは「位置」。

削除すると：

[A, B, C]
↓ B削除
[A, C]

Cのindexが変わる。

Reactは「別物」と誤認する可能性がある。



⑤ idが安全な理由

idは「正体」。

並び替えても変わらない

削除しても変わらない

追加しても変わらない

Reactが欲しいのは「位置」ではなく「正体」。

だから：

👉 key に一意なidを入れる



🎯 今日の核心

Reactのmapは

配列を

JSXに変換し

keyで正体を管理する

これだけ。
