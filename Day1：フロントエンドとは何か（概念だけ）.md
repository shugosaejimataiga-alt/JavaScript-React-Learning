🟦 Day1：フロントエンドとは何か（概念だけ）

今日は まだコードを書かない日 です。
フロントエンドを「何をする仕事なのか」という視点で整理していきます。

■ まず超シンプルに説明すると…

フロントエンド＝ユーザーが触る部分を作る仕事。
（ボタン、画面、入力フォーム、表示される情報など）

でも本質はもう少し深いです。



■ 本質的に大切な考え方
✔ 1) フロントは「画面をつくる」だけではない

フロントは、
ただ UI を並べる仕事ではなく、

“データがどう動き、それが画面にどう反映されるか” を設計する仕事
です。



✔ 2) バックエンドとは役割が違う

バックエンド：データを保存・取得・計算

フロント：データを受け取り、画面に反映して使える状態にする



✔ 3) React を学ぶ理由にもつながる

現代のアプリは「画面が動きまくる」。
→ 手作業で DOM を書き換えるのは破綻しやすい
→ React のような仕組みが必要になる

ここまでが今日の概念。

疑問： DOMとは何ですか？

✅ DOMとは何か（超シンプル版）
DOM = ブラウザが作る「HTMLをツリー状にしたデータ構造」

もっと短く言うと、

「ブラウザが理解できる、画面の設計図」
🟦 例でイメージする

HTML がこうなら：

<div>
  <h1>Hello</h1>
  <p>こんにちは</p>
</div>

ブラウザの中では、こんな “木のような構造” に変換される：

div
　└ h1
　　　└ "Hello"
　└ p
　　　└ "こんにちは"

これが DOM（Document Object Model）。



🟦 なぜ大事なの？

JavaScript が画面を触るときは、HTMLそのものを読むのではなく
この「DOMツリー」を触る。

例：

document.getElementById()
element.textContent = "変更！"
element.appendChild()

全部 DOM を操作している。



🟦 React の話につながるポイント（めちゃ重要）

DOM の操作は本来かなり重い。

だから…
手動で DOM を書き換える
どこを直すべきか自分で管理する
これは大規模になると 破綻する。

→ これが React（仮想DOM） の登場理由。


✔ DOM = ブラウザが HTML をツリー状のデータに変換したもの
「人間の認識」ではなく
　　　「ブラウザ（コンピュータ側）の認識の仕方」
