
## 1.今回の学習の説明（今日はここだけ）

今日のテーマ：
👉 「import / export は何のためにあるのか」


まず結論だけ言います。

import / export は
「ファイルを分けたあとも、コード同士をつなぐための仕組み」
です。



なぜ必要になった？

Phase2 までで、あなたはすでにこういう感覚を持っています：

JSが大きくなると
👉 1ファイルが長くなって辛い

処理ごとに分けたい
👉 役割ごとにファイルを分けたい


ここで問題が起きます。

ファイルを分けたら
他のファイルの関数や変数が使えなくなる

それを解決するのが export / import です。



役割を一言で

export
👉「このファイルのこれ、外で使っていいよ」

import
👉「あのファイルで公開されてたやつ、ここで使うね」

📌 公開（export）しないものは、外から見えない
これが超重要です。



たとえ話（かなり大事）

ファイル = 部屋

export = ドアに「入ってOK」と札を出す

import = その部屋に入って使う

👉 ドアがない（exportしてない）部屋には入れない。

## 2.基本の書き方（今日はここを理解する）

まず全体像（最初に結論）

流れはこれだけです。

使わせたい側 → export

使う側 → import



基本形①：export（外に出す）

// math.js
export const add = (a, b) => {
  return a + b;
};


意味を日本語にすると：

add という関数を定義した
export を付けた

👉 この関数は他のファイルから使っていい

📌 export がついていないものは
このファイルの中だけ専用




基本形②：import（取り込む）

// main.js
import { add } from "./math";

const result = add(2, 3);


意味：

"./math" ファイルから
{ add } を取り込む

👉 math.js で export された add を使っている




重要ポイント（超大事）

import { add } from "./math";

ここで：

{ add } ← 名前は一致必須

./math ← ファイル名（拡張子は省略可）

👉
export した名前と同じ名前で import する



ここで整理（1文で）

export は「公開」
import は「その公開物を使う」

## 3.練習問題

次のコードを見てください。

// user.js
export const userName = "Taro";
const age = 20;

// main.js
import { userName, age } from "./user";


質問（1つだけ）

この main.js は 正しく動きますか？
動く / 動かない のどちらですか？


解答：動かない。
理由はageはexportがついていない。という事は他のファイルでは使う許可が出ていないから

## 4.ここで超重要な一段深い理解


import / export がある本当の理由

単に「分けるため」ではありません。

目的はこれです：

外に見せたいものだけ見せる
それ以外は隠す
