🟦 Day5：DOMとは何か（概念だけ）

今日は まだコードを書かない日 です。
フロントエンドの基盤となる「DOM（ドム）」を、ブラウザの視点から整理します。

■ まず超シンプルに説明すると…

DOM ＝ ブラウザが HTML を理解するために作り直した “ツリー状のデータ構造”

あなたが書いた <div> や <h1> は、
ブラウザの内部では「ノード（枝）」のようなデータに変換され、
親子関係でつながった 1 本の巨大な木（Tree）になります。



■ 本質的に大切な考え方

✔ 1) DOM は「ブラウザが認識している世界」

私たちが書く HTML はあくまで 人間用の文章。
ブラウザはそのまま理解できないため、

→ HTML を ツリー構造のデータ（DOM）に翻訳
→ この DOM を使って画面を描画する

これが DOM の正体です。



✔ 2) DOM は JavaScript で “直接いじれる”

JavaScript は DOM を直接操作するための言語 です。

テキストを書き換える
要素を追加する
要素を削除する

これらはすべて DOM（ブラウザの内部データ）を直接変更しています。

つまり JavaScript は
「HTMLを触っている」のではなく、
「DOMというデータ構造を編集している」
ということです。



✔ 3) しかし DOM を直接いじるのは、人間にはしんどい

更新箇所が増えると複雑になる
UI の一貫性が崩れやすい
再描画の量が増えすぎると重くなる
保守性が低くなる

現代のアプリは要素が動きまくるため、
DOM を人間の手で管理するのは破綻しやすい。

この “辛さ” がそのまま React の必要性 につながります。



✔ 4) Reactは「DOM操作の中間管理職」

人間は

<h1>{title}</h1>

のように “意図” だけを書けばよく、
DOM をどう更新するかは React が肩代わり してくれます。

React は

仮想DOMを作り
差分を計算し
必要最小限の変更だけ本物の DOM に反映する

という仕事をすべて担当します。



■ 今日の理解ポイント（あなたが到達したレベル）

✔ DOM＝ブラウザがHTMLを理解した姿（ツリー構造）
✔ JavaScriptはDOMを直接操作できる
✔ DOM操作は複雑化しやすく、人間の管理には向かない
✔ Reactは“中間管理職”としてDOM操作を肩代わりしてくれる
✔ あなたが書くのは UI の意図だけで、DOM 更新は React が最適化
