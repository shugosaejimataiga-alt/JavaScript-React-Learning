🟦 Day49 ＋ イミュータブル思考 まとめ


① JSだけで作ると何が大変だった？

❓状態を変えたあと、どうしていた？
👉 render() を毎回手動で呼んでいた。

❓なぜそれが問題？
呼び忘れる
二重に呼ぶ
どこで更新されたか分からない
バグの温床になる
状態が増えるほど地獄になる。


② ではReactは何が楽？

❓Reactは何をしてくれる？
👉 状態を変えるだけで
👉 自動で必要な部分を更新してくれる

❓Reactが楽になる理由は？
「表示が楽」ではない。
👉 状態管理を安全にしてくれるから楽

再描画管理を自分でしなくていい
差分計算を自分でしなくていい
DOMを直接触らなくていい



🟦 イミュータブル思考

③ イミュータブルとは何？
👉 元のデータを直接変更しない考え方


❌ ミュータブル（悪い例）

user.name = "Jiro";

元データを書き換えている。


✅ イミュータブル（良い例）

const newUser = { ...user, name: "Jiro" };

新しいデータを作っている。


④ なぜReactで重要？

Reactは

👉 「中身」ではなく
👉 「参照が変わったか」を見ている


❓なぜ参照を見る？

中身チェックは重い
毎回全部比較すると遅い
参照なら一瞬で判定できる

だから

✔ 新しいものを作る → 変化検知できる
✖ 直接変更 → 変化検知できない可能性
