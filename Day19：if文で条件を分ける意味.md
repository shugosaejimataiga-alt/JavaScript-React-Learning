🟦 Day19：if文で条件を分ける意味（概念理解）

今日は if文を「文法」として覚える日ではなく、
なぜ存在するのかを理解する日 です。



■ まず超シンプルに説明すると…

if文＝「この条件のときだけ、実行していいかを判断する仕組み」

プログラムは基本的に
👉 上から順に、全部実行される。

そのままだと
👉 実行してはいけない処理まで動いてしまう。

それを防ぐために if文がある。



■ 本質的に大切な考え方

✔ 1) if文は「処理を書くもの」ではない

if文自体は、
何かを計算したり、表示を変えたりしない。

やっていることはただ一つ。

「この処理を、今やっていいか？」を判断しているだけ

つまり if文は
実行・非実行を分ける門番（安全装置）。


✔ 2) 条件は必ず true / false に分かれる

if文に必要なのは
✔ 条件（true / false になるもの）だけ。

true → 中の処理を実行

false → 中の処理は実行されない

これ以上でも、これ以下でもない。


✔ 3) フロントエンドでは if文がないと UI が壊れる

フロントエンドでは常に、

まだ入力されていない

データが空

状態が整っていない

という状況が発生する。

その状態で処理を走らせると
👉 画面が壊れる・エラーになる。

だから、

「この条件下なら、処理していい」
「この条件下では、まだダメ」

を分ける必要がある。

その役割を担うのが if文。


✔ 4) 状態（State）と if文は必ずセットで使われる

これまで学んだ
「状態（State）」と組み合わさることで、

状態を見る

条件を判断する

実行していい処理だけ通す

という流れが作れる。

👉 if文は、状態を安全に扱うための道具。



■ 今日の理解まとめ（1文）

if文は、条件を見て「していい処理」と「してはいけない処理」を分けるための仕組みである。
