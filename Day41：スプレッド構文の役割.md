## 2.基本の書き方（スプレッド構文）


配列の場合（いちばん大事）

const a = [1, 2, 3];
const b = [...a];


このとき何が起きているかというと：

a をそのままコピーしている ❌
a の中身（1,2,3）を取り出して、新しい配列を作っている ⭕


結果：

a === b; // false（別物）




追加したいとき（超よく使う）

const a = [1, 2, 3];
const b = [...a, 4];


意味は：

まず a の中身を全部並べる

その後ろに 4 を追加

新しい配列 b ができる

👉 a は一切変わらない




オブジェクトの場合（考え方は同じ）

const user = { name: "Taro", age: 20 };
const newUser = { ...user };


これも：

user を壊さない

中身を展開して

新しいオブジェクトを作る

日の学習の流れまとめ（Day41＋関連知識）

① Day41：スプレッド構文とは何か？

疑問
なぜスプレッド構文（...）を学ぶのか？

答え
元のデータを壊さずに
新しい配列・オブジェクトを作るため

Reactでは
👉「同じ箱を中身だけ変える」ではなく
👉「新しい箱を作ったか」で変化を判断する
そのためにスプレッド構文が必要。



② スプレッド構文の基本動作

疑問

const b = [...a] は何をしている？

答え
配列 a をコピーしているのではない

a の中身を展開して、新しい配列を作っている

const a = [1, 2];
const b = [...a, 3];
// a は変わらない
// b = [1, 2, 3]



③ 練習で確認したこと

疑問
どちらが変わる？

const nums = [10, 20];
const next = [...nums, 30];


答え
nums：変わらない
next：新しく作られる [10, 20, 30]

👉 変わるのは新しい方だけ



④ http / https とは何か？

疑問
http と https の違いは？

答え

やっていることは同じ（HTTP通信）

違うのは 暗号化されているかどうか

👉 https = http を安全にしたもの



⑤ CRUD とは何だったか？

疑問
「crod でしたっけ？」の正体は？

答え
👉 CRUD

Create（作る）

Read（読む）

Update（更新）

Delete（削除）



⑥ HTTPメソッドとCRUDの対応

GET → Read

POST → Create

PUT / PATCH → Update

DELETE → Delete

👉 Webアプリは
HTTP通信 → CRUD操作 → DB → JSON返却
という流れで動く。



⑦ https になると何が変わる？

疑問
https になると CRUD は変わる？

答え
👉 変わらない

GET / POST / CRUD / API
→ 全部同じ

通信の中身だけが暗号化される



⑧ ブラウザ保存とサーバ保存の違い

疑問
Cookie と データベース、どっちがどっち？

答え

ブラウザ側
Cookie
localStorage / sessionStorage

サーバ側
データベース（MySQL / PostgreSQL など）

👉 ユーザー情報の本体は DB に保存
