## 2.基本の書き方（ここが今日の核）

まず 全体像だけ 見せます。
※まだ覚えなくてOK、意味だけ掴みます。



1️⃣ export（外に出す）

別ファイルから使わせたいものに export をつけます。

// utils.js
export const add = (a, b) => {
  return a + b;
};


この時点で起きていることはこれだけ👇

add という関数を
このファイルの外でも使っていいよ
と宣言している



2️⃣ import（外から持ってくる）

さっき export したものを、別ファイルで使う

// main.js
import { add } from "./utils.js";

const result = add(2, 3);


意味を日本語にすると👇

./utils.js にある
add を
ここに持ってくる



3️⃣ 超重要な対応関係（ここだけ強調）

export const add = ...


⬇ 対になる ⬇

import { add } from ...


👉 名前は完全一致
👉 { } は「どれを持ってくるか」を指定している



今日覚えるのはこれだけ

export = 外に公開する

import = それを取り込む

分割したファイルを再び1つの仕組みに戻すため
