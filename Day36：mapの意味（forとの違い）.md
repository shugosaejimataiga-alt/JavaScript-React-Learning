## 2.基本の書き方（map）

まず結論だけ。


const numbers = [1, 2, 3];

const doubled = numbers.map((n) => {
  return n * 2;
});


結果はこうなります。

numbers  // [1, 2, 3]（変わらない）
doubled  // [2, 4, 6]（新しく作られた）



**何が起きているか（超重要）**

map は 配列専用

配列の 要素1つずつ を取り出す

returnした値だけ を集めて
👉 新しい配列を作る


これを一文で言うと：

map = 配列を「変換」して、新しい配列を作る



for文との決定的な違い

for文の場合：↓


const doubled = [];

for (let i = 0; i < numbers.length; i++) {
  doubled.push(numbers[i] * 2);
}




mapの場合：↓

const doubled = numbers.map(n => n * 2);


👉 やっていることは同じ
👉 mapは「配列変換専用に最適化された書き方」

📘 Day36｜mapの意味（forとの違い）

今日やったこと（結論から）

mapは配列を「変換」するための道具
元の配列は変わらない
新しい配列を必ず返す
for文と違い「配列 → 配列」専用の考え方



mapの正体（1文）

map = 配列の中身を1つずつ別の形にして、新しい配列を作る



今日使った具体例

const fruits = ["apple", "banana", "orange"];

const result = fruits.map(fruit => {
  return fruit + "!";
});


結果：

["apple!", "banana!", "orange!"]



mapの中で起きていること

mapは 配列専用
配列の中身を 1つずつ取り出す

取り出されるのは
👉 配列そのものではなく 1要素（今回は文字列）

return した値だけが集まって
👉 新しい配列 になる



for文との違い（考え方）

for文
→ 繰り返し処理の汎用ツール
→ 自分で配列を作って push する必要がある

map
→ 配列変換専用
→ 「どう変換するか」だけ書けばいい



今日出た疑問と答え

Q1. 元の配列は書き換わる？

A. 書き換わらない

mapは 元データを壊さない設計
表示用・加工用の配列を安全に作れる


Q2. mapの中に渡されるのは何？

A. 配列の1要素

例：

["apple", "banana", "orange"]


mapの中では順番に

"apple"
"banana"
"orange"

が渡される


Q3. なぜReactで重要？

A. データから画面を作るから

Reactは
👉 データが本体

mapは
👉 データ → 表示用構造 を作る最短ルート



今日の核心（これだけ覚えればOK）

mapは「配列を壊さずに、別の配列を作るための道具」
