🟦 Day15：変数（let / const）

今日は **JavaScriptで最初に触れる「状態の入れ物」**について学ぶ日。
まだアプリは作らず、考え方の理解が目的。

■ まず超シンプルに説明すると…

変数＝あとで使う値に「名前をつけて保存」するもの。

フロントエンドでは、
ユーザー操作や処理の途中結果など、
**変化していくデータ（状態）**を扱うために使う。



■ 今日学んだ中心テーマ

JavaScriptには主に2つの変数宣言がある。

let

const

この2つの違いを 「暗記」ではなく「意味」で理解することが重要。



■ 本質的に大切な考え方

✔ 1) let は「変わる前提の箱」

let は、あとで中身を書き換える可能性がある値に使う。

例：

カウンターの数
ユーザー操作で増減する値
状態（state）になりやすいもの

→ 再代入できるのが特徴。


✔ 2) const は「変えない前提の箱」

const は、一度決めたら変えない値に使う。

再代入はできない

「この値は固定」という意思表示

バグを防ぐ役割がある

→ 現代のJS / Reactでは
「まず const、必要なときだけ let」 が基本方針。


✔ 3) 判断基準は1つだけ

あとで代入し直すか？

する → let
しない → const

この基準だけで、ほぼ迷わなくなる。



■ 今日理解したことのまとめ

変数は「状態に名前をつける仕組み」

let は変化する状態用

const は変化しない前提の値用

フロントエンドでは「何が変わるか」を意識して変数を選ぶ

ここまでが Day15 の概念理解。
