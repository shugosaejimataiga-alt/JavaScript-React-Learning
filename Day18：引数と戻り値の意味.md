
## 1.基本

基本の書き方（最小）

**引数と戻り値のある関数**

function add(a, b) {
  return a + b;
}

a, b → 引数（外から渡すデータ）

return a + b; → 戻り値（結果）



**使う側（呼び出し）**

const result = add(2, 3);

console.log(result); // 5

関数は 値を返す

返った値は 変数に入れられる



**超重要ポイント（短く）**

console.log は 表示するだけ

return は 値を返す

Reactでは return が画面（JSX）になる

## 3.練習問題②（少しだけ発展）

次は 引数は1つ・条件付き です。

問題
次の条件を満たす関数を作ってください。

関数名：isAdult

引数：age

処理：

age が 20以上 → true
それ以外 → false
戻り値：判定結果（true / false）

期待する使い方
const result = isAdult(18);
console.log(result); // false

👉 コードだけ 書いて送ってください。
チェック後、Day18の最後の練習に進みます。


function isAdult(age) {
  return if (age >= 20) {
    console.log("true");
  } else {
    console.log("false");
  }
}

const result = isAdult(90);

console.log(result);



❌ 今回のズレ（結論）

return if (...) は JavaScriptでは書けません
console.log は 戻り値ではありません
この関数は true / false を返すのが目的です


**修正版**

function isAdult(age) {
  if (age >= 20) {
    return true;
  } else {
    return false;
  }
}

const result = isAdult(90);
console.log(result);

OKです。今のコードは完全に正解です 👍
if / return / 戻り値 の役割、ちゃんと分離できています。

## 4.練習問題③（Day18ラスト）

問題（引数＋条件＋戻り値）

次の条件を満たす関数を書いてください。

関数名：calcPrice

引数：price

処理内容：

price が 1000以上 → price * 0.9
それ以外 → price
戻り値：計算後の金額（数値）


期待する使い方
const result = calcPrice(1200);
console.log(result); // 1080

👉 コードだけ 書いて送ってください。

function calcPrice(price) {
  if(price >= 1000) {
    return price * 0.9;
  } else {
    return price;
  };
}

const calcPrice(3000);
console.log(calcPrice);


❌ 今回のミス（結論）
① 関数呼び出しは const 変数 = 関数() の形
② console.log には「値」を渡す


**修正版**

function calcPrice(price) {
  if(price >= 1000) {
    return price * 0.9;
  } else {
    return price;
  };
}

const result = calcPrice(3000);
console.log(result);

完璧です ✅
文法・考え方ともに100%正解です。

🟦 Day18：引数と戻り値の意味（概念理解）

今日は 関数の「中身」ではなく、「出入り口」 に注目する日です。
まだ複雑なコードは書きません。



■ まず超シンプルに説明すると…

引数＝関数に渡すデータ
戻り値＝関数が処理した結果として返すもの

関数は
「入力 → 処理 → 出力」
という構造を持つ。



■ 本質的に大切な考え方

✔ 1) 関数は「処理」より「関係」を定義している

関数は
「こういう処理をする」こと以上に、

「この入力に対して、どんな結果を返すか」
という 因果関係 を定義している。

引数が変われば、戻り値も変わる。


✔ 2) 引数は「人間の意図を渡す手段」

関数は人間が作るもの。
引数は、

「今回はこのデータで処理してほしい」
という人間側の意図を具体化したもの。

処理は同じでも、
対象（データ）を差し替えられるようにするために存在する。


✔ 3) 戻り値は「次に使うための出口」

関数は実行して終わりではない。
結果を 人間や他の処理が受け取って使う ために、戻り値がある。

戻り値があることで、
処理同士を安全につなげられる。


✔ 4) Reactでは「画面」が戻り値になる

Reactコンポーネントの正体は、

引数（state / props）を受け取り、
戻り値として JSX（画面の設計図）を返す関数

人間がやるのは

データ（state / props）をどう作るか
どんな JSX を返すか

まで。

その先（仮想DOM生成・差分計算・DOM操作）は
Reactの内部責務であり、直接触らない範囲。


✔ 5) 「電流」の比喩で理解できる

引数：流れ込む電流

関数：回路

戻り値：流れ出る電流

流れがあるから意味が生まれる。
Reactも同じ構造で動いている。



■ 今日のまとめ（Day18）

関数とは、
引数という入力を受け取り、
処理を通して、
戻り値という結果を返す装置。

Reactではこの戻り値が JSX（画面） になり、
画面更新はその結果として起きる。

ここまでが Day18で理解すべき概念 です。
