## 1.基本の書き方（DOM取得）


**HTML**

<p id="message">こんにちは</p>


ここで重要なのは id です。
JSは id を使って要素を特定します。



**JavaScript**

const message = document.getElementById("message");


これで起きていることを言葉にすると：

document
→ 画面全体（HTML全部）

getElementById("message")
→ 「id が message の要素を取ってきて」

const message =
→ それを 変数に入れて保持

つまりこの1行は：

「id が message の HTML 要素そのものを、JSで掴んだ」

という意味です。

⚠️ ここでは

表示は変わらない
中身も変えていない
ただ持ってきただけです。



**よくある勘違い（重要）**

const message = document.getElementById("message");
console.log(message);

これは 文字列を取っているわけではありません。

👉 HTML要素そのもの（オブジェクト）です。

**勘違いの例は結局どうなるのか？**

結論（まず答え）

👉 HTML要素そのものが表示されます。

**何が出るのか（中身）**

ブラウザのコンソールでは、だいたいこう表示されます：

<p id="message">こんにちは</p>


もしくは内部的には：

<p id="message">...</p>


これは 文字列ではありません。



**じゃあ「文字」を見たい場合は？**

ここではまだ 変更しません。
中身を見るだけです。

console.log(message.textContent);


これで初めて：

こんにちは

が出ます。

🟦 Day23：DOM取得とは何か（getElementById）

今日は **DOMを「操作する前段階」**として、
取得（参照）だけを理解する日でした。



■ まず超シンプルに説明すると…

DOM取得＝HTML要素を JavaScript から参照できる状態にすること。

JavaScript は
HTMLをそのまま勝手に触れるわけではなく、

「どの要素を扱うのか」を明示的に指定する必要がある。

その指定が DOM取得。



■ 今日理解した重要なポイント

✔ 1) getElementById が返すもの

document.getElementById() が返すのは、

文字列ではなく、HTML要素そのもの（DOMオブジェクト）。

const msg = document.getElementById("msg");

この時 msg に入っているのは
「こんにちは」ではなく、

<p id="msg">こんにちは</p>
という 要素全体。


✔ 2) 取得しただけでは何も変わらない

DOM取得は 参照なので、

表示は変わらない

中身も変わらない

「つかんだだけ」の状態。

表示や中身を変えるには、
次の指示（textContent など）が必要。


✔ 3) 要素と中身を分けて考える必要がある

DOM操作は常に2段階で考える。

1️⃣ どの要素か（DOM取得）
2️⃣ その要素のどこを扱うか（中身・属性・表示）

この区別ができていないと、

console.log でタグが出て混乱する

文字を直接代入しようとして詰まる

という問題が起きる。



■ 今日の理解を一言でまとめると

JavaScriptは、DOMを取得してから「どう扱うか」を指示する。

👉 「どれを扱うか」だけを学んだ日。
