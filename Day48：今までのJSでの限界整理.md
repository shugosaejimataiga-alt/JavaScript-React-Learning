📘 Day48：今までのJSでの限界整理


🎯 今日の目的

Reactを学ぶ前に、

「なぜ普通のJSでは限界が来るのか」

を言語化すること。



🧠 今までのJSの基本構造

let state = [];

function render() {
  // DOMを消す
  // stateを元にDOMを作る
}

function addItem(item) {
  state.push(item);
  render();
}


仕組み：

状態を持つ
状態が変わる
renderを手動で呼ぶ
DOMを作り直す

👉 小規模なら問題なし



🚨 JSの限界①：処理が追えなくなる

どこで state が変わった？
render は何回呼ばれた？
どの操作が原因？

コードが増えると
人間の頭で追えなくなる



🚨 JSの限界②：責務が混ざる

同じ場所に：

データ処理
DOM操作
イベント処理

が全部書かれる。

👉 修正が怖くなる
👉 バグが増える



🚨 JSの限界③：状態が増えると破綻

例：

dreams
actions
progress
表示モード
フィルター条件

状態が増えるほど、

「どの変更がどこに影響するか」

が分からなくなる。



💡 今日の核心

今までのJSは：

✔ 動く
✔ 作れる

でも、

❌ 大きくすると考え続けられない



🧩 今日の一文まとめ

普通のJSは動くが、規模が大きくなると人間の認知が限界を迎える。



🔥 Reactはなぜ必要？

Reactは速くするためではない。

人間が考え続けられる構造を作るため

に存在する。
