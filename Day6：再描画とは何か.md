🟦 Day6：再描画とは何か（概念だけ）

今日は 「画面が変化するとは何か」 を、
DOM・状態・React の理解につながる視点で整理していきます。

■ まず超シンプルに説明すると…

再描画＝DOM が書き換わり、画面が描き直されること。

ブラウザは HTML をもとに画面（UI）を描いているため、
HTML（DOM）が変わらない限り、画面は変わりません。



■ 本質的に大切な考え方

✔ 1) 画面の変化の本体は「DOM の変化」

ユーザーが見る UI は DOM の結果でしかありません。

状態（データ）が変わる
DOM に反映される
ブラウザがもう一度描き直す（＝再描画）

この流れが必ず起きています。



✔ 2) DOM を手で書き換えると破綻する

ボタン・リスト・テキスト・色・入力値など…

UI は “依存関係だらけ” です。

手作業で DOM を触ると：

更新漏れが出る
画面とデータがズレる
修正箇所が爆発する
規模が大きいほど壊れやすくなる

つまり 手動の DOM 操作は限界があるということです。



✔ 3) React が必要になる理由につながる

再描画の本質を理解すると…

→ 「DOM 操作を人間がやるのは無理」
→ 「状態だけ書けば UI が勝手に更新されてほしい」

という必然的なニーズが生まれます。

React はこの問題を解決するために、

仮想DOMで差分だけを検出し、最小限の DOM 更新を自動でやってくれる “中間管理職”

として動いてくれます。

これが 手動再描画が辛い → React が必要 という流れです。
