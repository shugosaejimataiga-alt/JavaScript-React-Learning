
基本の書き方（最小）

まず 最も基本形 だけ見ます。

function greet() {
  console.log("こんにちは");
}

これは

greet という名前の関数を定義しているだけ
まだ 実行はされていない

次に、呼び出しです。

greet();

これで初めて
こんにちは が表示されます。

function sayHello() {

関数を「2回」呼ぶことも可能

関数の強みは、
同じ処理を何度でも使えることです。

例（見るだけ）
sayHello();
sayHello();

→ 同じ表示が2回実行されます。



練習問題②

次の条件を満たしてください。

今書いた sayHello を 2回 呼び出す
関数の中身は変えない

👉 修正したコード全体 を貼ってください。

（確認できたら、次は「引数あり関数」へ進みます）


function sayHello() {
  console.log("Hello!");
}

sayHello();
sayHello();

完璧です。
**「定義は1回・呼び出しは何回でも」**がきちんと体で理解できています。

関数の中で処理を書く練習

関数は 1行だけ である必要はありません。

例（見るだけ）
function showMessage() {
  console.log("処理1");
  console.log("処理2");
}

呼び出すと、
中に書いた処理が 上から順に全部実行 されます。


練習問題③（Day17ラスト）

次の条件を満たす関数を書いてください。

関数名：startApp

中で

"初期化完了"

の2行を console.log で表示

最後に 1回だけ呼び出す

function startApp() {

startApp();

🟦 Day17：関数とは何か（定義・呼び出し）

今日は まだ本格的な実装に入らず、

関数の概念そのもの を整理する日です。

■ まず超シンプルに説明すると…
関数＝同じ処理を、名前を付けてまとめたもの。
何度も出てくる処理や、意味のある処理のかたまりを
「あとから呼び出せる形」にしておく仕組み。



■ 本質的に大切な考え方
✔ 1) 関数は「書いただけ」では動かない
関数には必ず

定義（用意する）

呼び出し（実行する）

の2段階がある。

呼び出されたときにしか処理は動かない。
「関数を書いたのに何も起きない」理由は、
→ 呼び出していないから。


✔ 2) 関数は「身軽な部品」
関数を使うことで、

使うときは呼ぶだけ

いらなくなったら消すだけ

修正は1か所で済む

という、人間側が楽になる構造 が作れる。
これは
「処理をどう書くか」より
「考えをどう分けるか」 の道具。


✔ 3) Reactにつながる重要な前提
この先のReactでは、

コンポーネント

画面の構造

処理の流れ

ほぼすべてが 関数ベース で成り立つ。
そのため、
「定義と呼び出しを分けて考える」
という理解が、React理解の土台になる。



■ 今日の理解の核心

関数は「処理のかたまり」

動かすのは常に「呼び出し」

関数はコード整理ではなく 思考整理の仕組み

ここまでが今日の概念。
