

## 1.今回の学習の説明（まずここだけ）

テーマ：アロー関数（=>）とは何か


結論だけ先に言います。

**アロー関数は「短く・意図をはっきり書くための関数の書き方」**です

特に map / filter / reduce と相性が良い
Reactでは ほぼ標準の書き方 になります


今までの関数は、こうでしたね👇

function add(a, b) {
  return a + b;
}


アロー関数は、これを別の形で書くだけです。
👉 できることは同じです。



なぜ新しい書き方が必要になったのか？

理由は1つだけです。
「関数の中身」より「何をしているか」を目立たせたい

numbers.map(function(n) {
  return n * 2;
});


これ、やっていることは単純なのに、
function や {} や return が目立ちますよね。

そこで出てきたのが アロー関数です。

## 2.基本の書き方（ここがDay39の核心）

基本形

const 関数名 = (引数) => {
  処理
};



例：1つの数を2倍する

const double = (n) => {
  return n * 2;
};



さらに省略できる形（重要）

return だけのときは、こう書けます👇

const double = (n) => n * 2;



意味は同じです。

{} を消す
return を消す
右側の式がそのまま返り値



アロー関数は「const と一緒に使うのが基本」

function は別の書き方（昔からある宣言方法）

アロー関数は function の代わりとして使うもの


「アロー関数は、function の代わりになる書き方で、
必ず const や let と一緒に使う」

## 3.問題（超基本）

次の function をアロー関数に書き換えてください。

function triple(n) {
  return n * 3;
}


条件：

const を使う
省略できるところは省略してOK

✍️ 答えを書いてください。



解答：

const triple = (n) => n * 3;


完璧です ✅
Day39：アロー関数の核心をしっかり押さえています。


なぜこの1行で正しいのか（確認）

const triple = (n) => n * 3;


① const triple

関数を再代入不可の変数として定義している
Reactでの標準的な書き方


② (n) =>

function(n) と同じ意味
引数が1つでも ( ) は付けてOK（安全）


③ n * 3

{} と return を省略
この式がそのまま戻り値



ここで重要な境界線（Day39の到達点）

アロー関数は 機能が増えるわけではない
「短く書ける」ことで、処理の意図が見えやすくなる
map / filter / reduce の読みやすさが激変する

## 4.次へ行く前のミニ確認（1問だけ）

次のうち、文法エラーになるものはどれでしょう？
（1つだけ）

A

const f = x => x + 1;



B

const f = (x) => {
  return x + 1;
};



C

const f = x => {
  x + 1;
};


答えを書いてください。



解答：AとC　理由は引数に()がないから

いいところまで来ています。結論は修正が必要です。



正解

❌ A はエラーではありません
⭕ C だけがエラーです



なぜか（端的に）


A（エラーではない）
const f = x => x + 1;


引数が1つだけのときは ( ) を省略できる

これは 正しいアロー関数



C（エラー）

  x + 1;
};


{} を使ったら 必ず return が必要

今は「計算して捨てている」状態 → 戻り値がない


正しくは👇

const f = x => {
  return x + 1;
};



**覚え方（重要）**

引数1つ → ( ) は省略OK

{} を使う → return 必須

1行で返すだけ → {} と return を省略できる

今日学んだこと（結論）

アロー関数は function の代わりになる「別の書き方」

機能が増えるわけではなく、短く・意図を明確に書くための記法

React・map / filter / reduce では 標準の書き方



基本の形

const 関数名 = (引数) => {
  処理
};


関数は 値 なので、const に入れて定義する。



省略ルール（最重要）

① 引数が1つだけ
x => x + 1

( ) は省略できる（エラーではない）



② return だけの処理
n => n * 2

{} と return を省略できる
右側の式が そのまま戻り値



③ {} を使ったら？
n => {
  return n * 2;
}

必ず return が必要
書かないと「計算して捨てるだけ」になる



function との違い（混乱しやすい点）

function

関数専用の古い宣言方法



アロー関数

const + 関数（値）
