🧠 Day53：コンポーネントとは何か（思考の流れ）


❓疑問1

画面を全部1つにまとめるのは良いのか？

💡答え

よくない。
大きくなると管理が大変になる。
→ パーツごとに分けたほうがよい。



❓疑問2

Reactはどうやって画面を作るのか？

💡答え

UIを「部品」に分けて組み立てる。



❓疑問3

その部品とは何か？

💡答え

画面の一部（JSX）を返す関数

例：

function Header() {
  return <h1>Hello</h1>;
}

これは
「HeaderというUI部品を定義している」



❓疑問4

どうやってその部品を使うのか？

💡答え
<Header />

と書く。

これは

「Header関数をUIとして呼び出す書き方」



❓疑問5

なぜ Header() ではなく <Header /> なのか？

💡答え

Reactは「HTMLっぽくUIを書く」思想だから。
JSXという仕組みで内部的に関数実行に変換される。



❓疑問6

<Header /> が3つあったら？

💡答え

3回実行される → 3回表示される。



🎯 今日の核心

コンポーネントとは？

👉 画面の一部を返す関数

Reactとは？

👉 部品を作って組み立てる仕組み
