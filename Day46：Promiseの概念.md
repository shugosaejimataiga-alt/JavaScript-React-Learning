

## 1.Day46｜今回の学習の説明（概念だけ）

Promiseとは何か？

Promiseは一言でいうと
「未来に結果が返ってくる処理を、約束として表現する仕組み」 です。

ここで大事なのは👇

まだ結果は出ていない
でも「成功するか」「失敗するか」は あとで必ず決まる

その状態を 値として扱える のが Promise



なぜPromiseが必要か

Day45でやった 非同期処理 では、

いつ終わるかわからない処理
終わった「あと」に何かしたい

という問題がありました。

Promiseはこの
「終わったあと何をするか」を、事前に書ける仕組み
です。



超重要な整理（ここだけ覚えてください）

Promiseは

処理そのもの ではない
未来の結果が入る箱

です。

中身はあとで

成功（resolve）
失敗（reject）

のどちらかになります。

## 2.基本の書き方（Promiseの形だけ）

まず 形だけ 見ます。意味は後で噛み砕きます。


const promise = new Promise((resolve, reject) => {
  // 非同期処理を書く
});


ここでの役割

new Promise(...)
→ Promiseという箱を作っている

(resolve, reject) => {}
→ Promise専用の関数

resolve：成功したときに呼ぶ

reject：失敗したときに呼ぶ



超重要ポイント（ここだけ）

Promiseの中では
「いつか resolve するか reject するか」を決める

まだ結果は返っていない

でも 「返し方」は決まっている


例を一つだけ👇

const promise = new Promise((resolve, reject) => {
  resolve("OK");
});


これは
👉 「このPromiseは成功して、'OK' を持つ」
と約束しているだけです。

※ この時点では

画面は変わらない
consoleにも出ない

ただの「箱」

## 3.練習問題

問題1

次のコードを見てください。

const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve("完了");
  }, 1000);
});

promise.then(result => {
  console.log(result);
});


質問です👇
👉 「完了」 が console に出るのは、どのタイミングですか？

1.new Promise が実行された瞬間

2.setTimeout が終わって resolve が呼ばれた後

番号で答えてください。



解答：2.setTimeout が終わって resolve が呼ばれた後


正解です 🎯
理解はとてもきれいです。
