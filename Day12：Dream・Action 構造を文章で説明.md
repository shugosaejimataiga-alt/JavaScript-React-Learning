🟦 Day12：Dream / Action 構造を文章で説明
① 今日のテーマ

人生ToDoアプリの中身（データ構造）を、人に説明できるレベルまで言語化すること。



② Dream と Action の関係（超重要）

Dream
人生の「目的・到達点」。

例：

ITエンジニアになる
健康的な生活を送る


Action
Dream を実現するための「具体的な行動」。

例：

JavaScriptを勉強する
毎日30分運動する

👉 関係性はこれだけ

1つの Dream に、複数の Action がぶら下がる



③ なぜ分けるのか（設計の理由）

Dream だけだと「願望」で終わる

Action だけだと「何のための行動か分からなくなる」

だから
「意味（Dream）」と「行動（Action）」を分離する

これはフロント設計でも、
バックエンド設計でも、
ほぼ必ず出てくる考え方。



④ フロント視点での理解（まだコード不要）

画面には

Dream の一覧がある

Dream を押すと、その Dream に紐づく Action が見える

つまりフロントは
「親（Dream）→ 子（Action）」の構造を表示しているだけ
