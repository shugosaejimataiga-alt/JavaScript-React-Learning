・オブジェクトの基本の書き方（次にこれ）

const user = {
  name: "guest",
  age: 20
};

意味（端的に）

{} は「オブジェクト」

name や age は キー

1つのものの「性質」をまとめる



配列 × オブジェクト（Day16の核心）
基本の書き方（まず例を見るだけ）
const users = [
  { name: "あきら", age: 35 },
  { name: "さとし", age: 28 }
];


意味（端的）

users は配列

中身は オブジェクト

「人が複数いる状態」を表せる

🟦 Day16：配列・オブジェクトとは何か（概念だけ）

「アプリのデータはどう整理されるべきか」という視点で考えます。



■ まず超シンプルに説明すると…

配列：同じ種類のデータをまとめる箱

オブジェクト：1つのものが持つ、複数の情報をまとめる箱



■ 本質的に大切な考え方

✔ 1) 変数だけではアプリは成り立たない

変数はデータを1つしか持てない。
しかしアプリでは、

夢は1つではなく複数ある

Actionも複数ある

→ 同じ種類のデータを「集合」として扱う必要が出てくる。


✔ 2) 配列は「同じ種類の集合」を扱うための考え方

配列の本質は、
同じ種類のデータを、まとめて処理できるようにすること。

1つずつ個別に扱うのではなく、
「全体に対して同じ処理をする」ために必要になる。


✔ 3) オブジェクトは「1つのもの」を表すための考え方

1つの夢には、

タイトル
理由
進捗

など、性質の異なる情報が複数ある。
それらを「1つの意味あるまとまり」として扱うために
オブジェクトという考え方が必要になる。


✔ 4) アプリの基本構造はこの組み合わせ

夢1つ → オブジェクト

夢が複数 → オブジェクトの配列

つまり、
配列の中にオブジェクトが入る
これがアプリのデータ構造の基本形。


✔ 5) 今は「書き方」を知らなくていい

この段階の目的は、

なぜ配列・オブジェクトが必要か
何をまとめるためのものか

を 言葉で説明できるようになること。
コードの形は、後から自然に理解できる。
